# Artifact Contracts

Use these templates as contracts, not mandatory bureaucracy. Preserve an existing equivalent artifact when it already serves the same purpose.

## README.md

Purpose: one-minute project entry point.

Minimum fields:

```markdown
# Project title

## Purpose
## Current research question
## Current phase
## Repository map
## Reproduce or run
## Key artifacts
## Contacts or ownership
```

## PROJECT.md

Purpose: stable scientific intent.

```markdown
# Project specification

**Status:** provisional | approved | superseded
**Last updated:** YYYY-MM-DD

## Research problem
## Current state of knowledge
## Knowledge gap
## Research question
## Objectives
## Hypotheses or predictions
## Scope
### Included
### Excluded
## Study system
## Available data
## High-level approach
## Validation strategy
## Expected outputs
## Open questions
```

## STATUS.md

Purpose: current state and handoff. Keep history in Git.

```markdown
# Project status

**Last updated:** YYYY-MM-DD
**Current phase:**
**Current objective:**
**Branch or commit:**

## Completed
## In progress
## Next
## Blockers
## Known uncertainties
## Authoritative inputs
## Latest valid outputs
## Read first
```

## METHODS.md

Purpose: current planned and implemented methods.

```markdown
# Methods specification

**Document status:** evolving
**Last updated:** YYYY-MM-DD

## Study design
## Data sources
## Eligibility and sampling
## Data preparation
## Variables
## Analysis
## Validation and robustness
## Software and environment
## Deviations from initial plan
## Implementation map
```

Each subsection may include:

```markdown
**Status:** planned | implemented | validated
**Implemented in:**
**Configuration:**
**Inputs:**
**Outputs:**
```

## DECISIONS.md

Purpose: consequential reasoning history.

```markdown
# Decision log

## D-001: Decision title

**Date:** YYYY-MM-DD
**Status:** proposed | accepted | superseded
**Supersedes:** [ID or none]

### Context
### Decision
### Alternatives considered
### Reason
### Consequences
### Evidence
### Implementation
```

Record a decision only when reasonable alternatives existed or the choice materially affects validity, interpretation, reproducibility, cost, or scope.

## AGENTS.md

Purpose: project-specific operating rules.

```markdown
# Agent instructions

## Project objective
## Read first
## Sources of truth
## Working rules
## Data restrictions
## Commands
## Validation
## Writing boundaries
## Update requirements
```

Keep it short. Point to other artifacts rather than copying them.

## data/README.md

Purpose: provenance and data semantics.

```markdown
# Data documentation

## Sources
| ID | Dataset | Provider | Version/access date | Coverage | Resolution | Licence/access |

## Folder rules
## Data flow
## Variable dictionary
## Missing values
## Sensitive or restricted data
## Dataset versions
## Processing map
```

Prefer a machine-readable data dictionary when there are many variables.

## config/analysis.yml

Purpose: authoritative analysis parameters.

```yaml
project:
  id: ""
  analysis_version: ""

data:
  input_version: ""
  processed_version: ""

sampling: {}
processing: {}
model: {}
validation: {}

reproducibility:
  random_seed:
  environment_lockfile:
```

Do not place narrative justification here.
