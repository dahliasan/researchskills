# Mode: response-to-reviewers

## Steps

1. Inventory each reviewer comment (or comment group) with an ID.
2. For each item, decide: agree / clarify / disagree — with evidence.
3. Apply evidence and conflict gates to every manuscript change you propose.
4. Prefer surgical manuscript edits ([surgical-edit.md](surgical-edit.md))
   unless the comment requires structural revision
   ([revise.md](revise.md) or [draft.md](draft.md)).
5. Build a response map: comment → response → manuscript locus → change
   made or rationale for no change.
6. Load [scaffolding.md](../scaffolding.md) before returning revised Methods
   or Results prose.
7. Do not invent new analyses in the response letter; route missing analysis
   work to `research-project-ops` / analysis owners.

**Done when** every comment has a mapped response, manuscript changes are
gated, and the response letter does not claim work that was not done.

## Output

1. Response map (comment → reply → manuscript change or justification)
2. Revised manuscript spans
3. Material `[TBC]` / conflicts / deferred analysis items listed separately
